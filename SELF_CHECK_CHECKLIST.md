# AI 自检清单

## 自检频率
- 每次代码提交前
- 每次功能更新后
- 每次依赖升级后

## 自检流程

### 1. 架构分层检查
- [ ] 原子层是否无依赖其他层
- [ ] 模板层是否只依赖原子层
- [ ] 数据层是否只依赖原子层
- [ ] 业务层是否只依赖模板层、数据层、原子层
- [ ] 应用层是否只依赖业务层、数据层、原子层
- [ ] 同层模块之间是否无相互依赖

### 2. 原子化检查
- [ ] 所有枚举是否都是字符串类型
- [ ] 所有枚举是否都包含 UNKNOWN 兜底值
- [ ] 所有接口是否都继承 IBase
- [ ] 所有游戏规则是否都已原子化拆解
- [ ] 是否无硬编码规则（所有规则都可配置）

### 3. 鲁棒性检查
- [ ] 所有函数入口是否都有参数校验
- [ ] 所有外部数据是否都经过 sanitization
- [ ] 所有数值输入是否有边界检查
- [ ] 所有字符串输入是否有长度限制
- [ ] 所有选择输入是否有兜底值
- [ ] 所有异步操作是否都有 try-catch
- [ ] 是否使用错误边界组件

### 4. 代码规范检查
- [ ] 是否使用 2 个空格缩进
- [ ] 每行代码是否不超过 120 个字符
- [ ] 所有语句是否以分号结尾
- [ ] 变量和函数是否使用小驼峰命名
- [ ] 接口和类型是否使用大驼峰命名
- [ ] 常量是否使用全大写 + 下划线命名
- [ ] 文件是否使用短横线命名

### 5. TypeScript 检查
- [ ] 是否启用严格模式
- [ ] 是否无 any 类型使用
- [ ] 是否无类型断言（除非绝对必要）
- [ ] 所有变量和函数是否有明确类型注解
- [ ] 运行 `npm run type-check` 是否无错误

### 6. 功能检查
- [ ] 卡牌类型选择是否正常
- [ ] 基本信息填写是否正常
- [ ] 技能效果编辑是否正常
- [ ] 实时预览是否正常
- [ ] PNG 导出是否正常
- [ ] 标记系统是否正常工作
- [ ] 配置管理是否正常工作
- [ ] 规则校验是否正常工作

### 7. 数据格式检查
- [ ] 导出的卡牌数据是否包含所有必要字段
- [ ] 导出的数据是否符合 JSON Schema
- [ ] 所有数据是否都包含版本号
- [ ] 自定义标记是否正确导出

### 8. 性能检查
- [ ] 列表渲染是否使用稳定的 key
- [ ] 纯组件是否使用 React.memo 包装
- [ ] 回调函数是否使用 useCallback 包装
- [ ] 计算结果是否使用 useMemo 缓存
- [ ] 是否避免在渲染函数中创建新对象

### 9. 安全检查
- [ ] 是否无 XSS 漏洞
- [ ] 是否无 SQL 注入风险
- [ ] 是否无敏感信息泄露
- [ ] 所有用户输入是否都经过校验

## 自动化检查命令

```bash
# TypeScript 类型检查
npm run type-check

# ESLint 代码检查
npm run lint

# 构建测试
npm run build

# 开发模式测试
npm run dev
```

   - [ ] 添加消耗模块

   - [ ] 删除技能
   - [ ] 检查承接主体传递

   - [ ] 查看内置标记列表
   - [ ] 创建自定义标记

   - [ ] 删除自定义标记

5. **配置管理测试**
   - [ ] 查看当前配置

   - [ ] 修改卡组大小上限
   - [ ] 导出配置
   - [ ] 导入配置

7. **错误处理测试**
   - [ ] 输入极端值（负数、超大数）

   - [ ] 输入超长字符串
   - [ ] 检查是否有崩溃或白屏
   - [ ] 检查错误提示是否友好

8. **数据持久化测试**
   - [ ] 创建卡牌后刷新页面
   - [ ] 检查数据是否保留
   - [ ] 导出卡牌数据文件
   - [ ] 导入卡牌数据文件
   - [ ] 检查数据是否正确恢复

## 问题修复优先级

### P0 - 严重问题（必须立即修复）
- 程序崩溃或白屏
- 数据丢失
- 安全漏洞
- 类型检查错误

### P1 - 高优先级（尽快修复）
- 功能不正常
- 校验规则失效
- 错误提示不明确
- 性能问题严重

### P2 - 中优先级（计划修复）
- UI 显示问题
- 用户体验不佳
- 代码规范问题
- 文档缺失

### P3 - 低优先级（有空修复）
- 代码优化
- 重构建议
- 功能增强
- 性能微调

1. 记录发现的问题

## 守则遵守检查

- [ ] 鲁棒性第一原则：所有输入都有校验和容错
- [ ] 原子化脱耦合原则：所有规则都已原子化
- [ ] 可配置性原则：所有参数都可配置
- [ ] 标准化原则：遵循统一的命名和格式规范
- [ ] 依赖规则：严格遵守单向依赖
- [ ] 错误处理：所有错误都被捕获和记录
- [ ] 数据不可变：所有状态更新都使用不可变方式
- [ ] 组合优于继承：无继承使用
