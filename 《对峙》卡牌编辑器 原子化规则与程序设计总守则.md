# 目标选择系统 原子化设计规范（补充）
**版本：v2.0 | 生效日期：2026-05-29 | 所属章节：第三部分 核心模块原子化设计规范**

目标选择系统是整个编辑器的核心基础模块，所有效果、条件、消耗都必须通过本系统获取目标。本系统严格遵循**原子化拆解、可配置扩展、鲁棒性优先**原则，将复杂的目标选择逻辑拆解为**目标类型+过滤器+排序器**三个独立可组合的原子单元，支持用户在规则边界内自由组合出任何目标选择逻辑。

---

## 一、目标系统核心架构（严格分层）
```
┌─────────────────────────────────────┐
│ 目标选择器 (TargetSelector)         │  对外统一接口
│  输入：目标配置对象                 │
│  输出：符合条件的目标列表           │
├─────────────────────────────────────┤
│ 目标类型解析器 (TargetTypeResolver) │  解析基础目标类型
│  输出：初始目标池                   │
├─────────────────────────────────────┤
│ 目标过滤器链 (TargetFilterChain)    │  按顺序执行所有过滤器
│  输入：初始目标池                   │
│  输出：过滤后的目标池               │
├─────────────────────────────────────┤
│ 目标排序器 (TargetSorter)           │  对目标池进行排序
│  输入：过滤后的目标池               │
│  输出：最终目标列表                 │
└─────────────────────────────────────┘
```

**强制规则**：
- 任何效果/条件/消耗都不能直接操作目标，必须通过`TargetSelector`获取
- 目标选择逻辑完全由配置决定，禁止在代码中硬编码任何目标选择规则
- 所有目标操作都是纯函数，相同输入必须产生相同输出
- 当目标池为空时，返回空数组，不抛出错误

---

## 二、基础目标类型枚举（不可扩展核心）
所有基础目标类型是系统内置的不可扩展原子单元，覆盖游戏中所有可能的目标集合。

```typescript
export enum TargetType {
  // 单目标
  SELF = "self",                    // 自己
  ONE_ENEMY = "one_enemy",          // 任意一名敌方
  ONE_ALLY = "one_ally",            // 任意一名友方
  ONE_UNIT = "one_unit",            // 任意一名单位
  DAMAGE_SOURCE = "damage_source",  // 伤害来源
  TRIGGER_UNIT = "trigger_unit",    // 触发者
  INHERIT = "inherit",              // 承接主体（来自条件）
  
  // 多目标
  ALL_ENEMIES = "all_enemies",      // 所有敌方
  ALL_ALLIES = "all_allies",        // 所有友方
  ALL_UNITS = "all_units",          // 所有单位
  ALL_ALIVE = "all_alive",          // 所有存活单位
  ALL_DEAD = "all_dead",            // 所有阵亡单位
  
  // 位置相关
  ADJACENT = "adjacent",            // 所有接壤单位
  ADJACENT_ENEMY = "adjacent_enemy",// 所有接壤敌方
  ADJACENT_ALLY = "adjacent_ally",  // 所有接壤友方
  LEFT_ADJACENT = "left_adjacent",  // 左侧接壤单位
  RIGHT_ADJACENT = "right_adjacent",// 右侧接壤单位
  
  // 特殊
  RANDOM_ENEMY = "random_enemy",    // 随机一名敌方
  RANDOM_ALLY = "random_ally",      // 随机一名友方
  RANDOM_UNIT = "random_unit",      // 随机一名单位
  LOWEST_LIFE = "lowest_life",      // 生命最低的单位
  HIGHEST_LIFE = "highest_life",    // 生命最高的单位
  LOWEST_ARMOR = "lowest_armor",    // 护甲最低的单位
  HIGHEST_ARMOR = "highest_armor",  // 护甲最高的单位
  LOWEST_ENERGY = "lowest_energy",  // 技力最低的单位
  HIGHEST_ENERGY = "highest_energy",// 技力最高的单位
  
  // 兜底值（鲁棒性保障）
  UNKNOWN = "self"
}
```

**鲁棒性规则**：
- 所有目标类型必须有明确的兜底值，当无法解析时默认返回`self`
- 当目标类型为单目标但目标池为空时，返回空数组
- 当目标类型为多目标但目标池为空时，返回空数组
- 阵亡单位自动从所有目标池中排除，除非明确指定`ALL_DEAD`

## 三、目标过滤器（可扩展原子单元）
目标过滤器是对初始目标池进行二次筛选的原子操作，支持任意数量的过滤器链式组合，逻辑关系默认为`AND`。

### 3.1 内置过滤器清单（必须实现）
| 过滤器ID | 功能描述 | 参数 |
|----------|----------|------|
| `filter_by_attribute` | 按属性值过滤 | attribute(属性), operator(比较符), value(数值) |
| `filter_by_mark` | 按标记过滤 | mark(标记类型), operator(比较符), count(数量) |
| `filter_by_card_count` | 按手牌数量过滤 | operator(比较符), count(数量) |
| `filter_by_card_type` | 按手牌类型过滤 | cardType(卡牌类型), operator(比较符), count(数量) |
| `filter_by_relation` | 按与承接主体的关系过滤 | relation(关系类型) |
| `filter_by_distance` | 按距离过滤 | operator(比较符), distance(距离) |
| `filter_by_alive` | 按存活状态过滤 | isAlive(布尔值) |
| `filter_by_equipped` | 按装备状态过滤 | equipmentType(装备类型) |

### 3.2 过滤器数据结构
```typescript
export interface ITargetFilter {

  params: Record<string, any>;
  operator: "AND" | "OR" | "NOT"; // 与上一个过滤器的逻辑关系

// 目标选择配置对象
export interface ITargetConfig {

## 四、目标排序器（可扩展原子单元）
目标排序器用于对过滤后的目标池进行排序，决定多目标效果的执行顺序。

### 4.1 内置排序器清单（必须实现）
| 排序器ID | 功能描述 | 参数 |
|----------|----------|------|
| `sort_by_attribute` | 按属性排序 | attribute(属性), order(asc/desc) |
| `sort_by_mark` | 按标记数量排序 | mark(标记类型), order(asc/desc) |

## 五、承接主体系统标准化
承接主体系统是目标选择系统的重要组成部分，用于在条件和效果之间传递目标信息，避免重复配置。

### 5.1 承接主体数据结构
```typescript
// 扩展 SkillEffect 接口
export interface ISkillEffect {
  id: string;
  conditions: IConditionInstance[];
  costs: ICostInstance[];
  effects: IEffectInstance[];
  
  // 承接主体存储
  inheritedTargets: {
    conditions?: Record<string, ITargetConfig>; // 条件产生的承接主体
    effects?: Record<string, ITargetConfig>;    // 效果产生的承接主体
  };
}
```

### 5.2 承接主体工作流程
1. **条件阶段**：当条件的目标配置中`saveAsInherit`为`true`时，将解析后的目标保存到`inheritedTargets.conditions`中
2. **效果阶段**：当效果的目标类型为`INHERIT`时，从`inheritedTargets`中获取对应的目标
3. **链式承接**：效果也可以将自己的目标保存为承接主体，供后续效果使用

### 5.3 承接主体配置示例
```typescript
// 示例：条件选择一名生命≥5的敌方，效果对该敌方造成伤害
const skillEffect: ISkillEffect = {
  id: "effect_example",
  conditions: [
    {
      conditionId: "cond_check_attribute",
      params: {
        target: {
          type: TargetType.ONE_ENEMY,
          filters: [],
          saveAsInherit: "main_target" // 保存为承接主体
        },
        attribute: AttributeType.LIFE,
        operator: ComparisonOperator.GTE,
        value: 5
      }
    }
  ],
  costs: [],
  effects: [
    {
      effectId: "eff_deal_damage",
      params: {
        target: {
          type: TargetType.INHERIT,
          inheritKey: "main_target" // 使用承接主体
        },
        damageType: DamageType.PHYSICAL,
        value: 3
      }
    }
  ],
  inheritedTargets: {}
};
```

## 八、鲁棒性保障措施
1. **空值处理**：当目标池为空时，返回空数组，不执行任何效果
2. **非法参数处理**：当过滤器参数非法时，跳过该过滤器，继续执行后续过滤器
3. **承接主体缺失处理**：当承接主体不存在时，默认使用`self`作为目标
4. **循环引用检测**：检测承接主体的循环引用，避免无限递归
5. **目标数量限制**：当目标数量超过限制时，只返回前N个目标
6. **阵亡单位自动排除**：除非明确指定，否则阵亡单位自动从目标池中排除
